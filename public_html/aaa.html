How to do AAA testing

AAA Testing

  1. Arrange-Act-Assert is a great way to structure test cases. It prescribes an order of 3 operations/steps.
  2. The arrange step sets up the test’s input values.
  3. The act step prompts the primary function being tested.
  4. The assert step verifies that the output of the function is what was expected.
  5. To think in technical terms, the AAA testing structure is as follows: assert(act(arrange()));
  6. Writing tests is essential to be able to maintain clean and usable code. It is necessary for preventing problems with your code later on. A sufficiently large and complex code base will inevitably contain bugs and issues. Keeping these bugs and issues to a minimum will largely depend on the comprehensiveness of your testing. It will also help when adding new functionality or refactoring your code, making sure you haven’t broken anything you didn’t intend to.
