Seperation of Concerns

  • The initial calculator program had all methods and functions in a single class. While did not have any significant outcome on functionality, it would be bad practice on large scale programs which are more prone to bugs and also bad curtesy to whoever reads your code.
  • The new calculator program demonstrates the seperation of concerns by distributing calculator functions across different classes.
  • For example, addition and subtraction are all in their own class. This way, any additional features or changes made to addition will not affect subtraction.
  • In addition, havinging a seperate class, history for example, helps with readability and debugging as all of the history related code is in this own class.
  • Having related methods grouped together can also assist with readability.
